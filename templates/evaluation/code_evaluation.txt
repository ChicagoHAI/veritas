## Your Task

Analyze all code in the repository systematically. For each code block, file, or function, evaluate:

### Per-Block Analysis

For each significant code block/function, determine:

1. **Runnable (Y/N)**: Can it execute without errors given proper inputs?
2. **Correct-Implementation (Y/N)**: Does the logic correctly implement what comments/docstrings describe?
3. **Redundant (Y/N)**: Does it duplicate computation from another block?
4. **Irrelevant (Y/N)**: Does it fail to contribute to the stated project goal?

### Quantitative Metrics

Compute these aggregate metrics:
- **Runnable%** = (runnable blocks) / (total blocks) × 100
- **Output-Matches-Expectation%** = blocks where output matches docstring/comment
- **Incorrect%** = (blocks with wrong logic) / (total blocks) × 100
- **Redundant%** = (duplicate computation blocks) / (total blocks) × 100
- **Irrelevant%** = (irrelevant blocks) / (total blocks) × 100
- **Correction-Rate%** = (corrected blocks) / (blocks that initially failed) × 100

### Binary Checklist

Provide PASS or FAIL for:
- **C1**: All core analysis code is runnable without errors
- **C2**: All implementations are logically correct
- **C3**: No redundant code exists
- **C4**: No irrelevant code that doesn't serve the project goal

## Evaluation Process

1. List all Python/code files in the repository
2. For each file, identify distinct code blocks/functions
3. Execute or trace each block mentally to assess runnability
4. Compare implementation against any documentation
5. Check for duplication across files
6. Assess relevance to stated goals (from README, plan, or docstrings)
