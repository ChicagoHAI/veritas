## Your Task

Analyze the consistency between:
- Code implementation and its documentation
- Reported results and actual outputs
- Plan/methodology and implementation
- Claims and supporting evidence

## Checklist Items

Evaluate each item as PASS or FAIL:

### CS1: Results vs Conclusion
All evaluable conclusions in the documentation (README, REPORT.md, etc.) must match what the code actually produces.
- **PASS**: Every stated result can be traced to code that produces that output
- **FAIL**: Any conclusion is unsupported by or contradicts actual code output

### CS2: Implementation Follows Plan
If a plan exists, implementation should reflect the planned steps.
- **PASS**: Each plan step has corresponding implementation
- **FAIL**: Plan steps are missing or implemented differently without explanation

### CS3: Effect Size
Reported effects should be non-trivial relative to baseline or natural variability.
- **PASS**: Results show meaningful differences, not noise
- **FAIL**: Effects are within expected random variation or trivially small

### CS4: Justification
Key design choices and intermediate conclusions should be explicitly justified.
- **PASS**: Major decisions (architecture, hyperparameters, etc.) have documented rationale
- **FAIL**: Important choices are made without explanation

### CS5: Statistical Significance
Key results should report uncertainty measures or statistical tests.
- **PASS**: Results include confidence intervals, p-values, or multiple runs
- **FAIL**: Single point estimates without uncertainty quantification

## Evaluation Process

1. Read all documentation files (README, REPORT, plan.md, etc.)
2. Identify all claims and stated results
3. Trace each claim to supporting code/data
4. Check for contradictions or unsupported statements
5. Verify effect sizes are meaningful
6. Check for statistical rigor
