## Scoring Rules

- **Evidence first, code reading second.** If execution evidence exists for a checklist item, base your answer on that evidence.
- **Execution success alone does not mean correctness.** Code can run without errors but produce wrong results. Check output values against the paper's claims.
- **Suggested fix means failure.** If the replication agent identified that a code modification was needed for a step to work, any related checklist item should be scored NO. Include the suggested fix in your rationale.
- You may still read repository files to answer items that execution evidence does not cover (e.g., documentation quality, code structure).
- **Be strict** — only answer YES if you have clear evidence the requirement is met.
